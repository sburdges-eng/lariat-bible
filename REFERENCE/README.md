# Reference Directory

## Read-Only Reference Materials

This directory contains source materials that should be treated as **read-only references**. These files are the foundation for recipe scaling, ingredient calculations, and vendor ordering.

## Files (To Be Added)

### Recipe_Book.docx
**Purpose**: Complete recipe database with yields and ingredient lists

**Contains**:
- All recipes with exact measurements
- Sauces, brines, and rubs
- Cooking instructions and techniques
- Yield information per recipe
- Scaling guidelines

**Usage**: Reference when calculating ingredient needs for events

---

### Order_Guide.xlsx
**Purpose**: Vendor products, pricing, and order codes

**Contains**:
- Product lists by vendor
- Current pricing
- Order codes/SKUs
- Pack sizes
- Vendor contact information

**Usage**: Reference when creating ingredient orders

---

### Ingredients_Master_List.xlsx
**Purpose**: Complete ingredient database (170+ items)

**Contains**:
- Full ingredient list
- Unit measurements
- Vendor sources
- Storage requirements
- Shelf life information

**Usage**: Reference for ingredient availability and sourcing

---

## How to Use This Directory

1. **When Planning an Event**:
   - Look up recipes in Recipe_Book.docx
   - Scale quantities based on guest count
   - Cross-reference ingredients with Master List

2. **When Ordering**:
   - Use Order_Guide.xlsx to find product codes
   - Check current pricing
   - Verify pack sizes and minimums

3. **When Calculating Costs**:
   - Reference pricing from Order_Guide
   - Factor in yield percentages from recipes
   - Account for waste and buffer (typically 10%)

## Maintenance

### Updating Reference Files

**Recipe Book**:
- Update when recipes are refined
- Document any yield changes
- Add new items as menu expands
- Version control: date-stamped backups

**Order Guide**:
- Update pricing monthly or when notified
- Add new products as discovered
- Archive old versions with dates
- Note any discontinued items

**Master List**:
- Add ingredients as needed
- Update vendors if sources change
- Document seasonal availability
- Review quarterly for accuracy

## Important Notes

- **Do not delete** old versions - archive them
- **Document changes** with dates and reasons
- **Test recipes** before events when updated
- **Verify pricing** before finalizing client invoices
- **Back up** these files regularly
